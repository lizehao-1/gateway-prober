Gateway Prober 详细说明

1. 本地和线上怎么用

如果你要测本地 `http://127.0.0.1:端口`、`localhost:端口`，请直接用仓库里的 Flask 版，因为 Cloudflare Pages 线上版只能访问公网 https 地址。

  • 本地 Flask 版：适合测本地部署、局域网、需要更灵活调试的情况。
  • Cloudflare Pages 版：适合把网页挂到域名上，给自己或别人远程使用。

2. 为什么很多时候 Base URL 要写到 /v1

很多兼容网关实际上只在 /v1 下暴露接口。遇到根地址测不出来、写到 /v1 才有结果时，通常不是你的 Key 有问题，而是这个网关本来就是这样设计的。

3. 如果 Embeddings 不可用，会影响什么

/v1/embeddings 不是聊天接口，而是把文本转成向量。它做不了时，普通聊天、代码问答、工具调用通常仍然可以用，但依赖“向量检索”的功能会变弱，甚至直接不可用。

  • 通常还能做：普通对话、代码补全、代码解释、Agent 调工具、基于 /chat/completions 或 /responses 的常规 IDE 助手。
  • 通常会受影响：知识库问答、RAG、项目语义搜索、文档召回、相似内容匹配、先检索再回答的工作流。

4. 常见后缀分别是干什么的

后缀用途说明
/v1/chat/completions传统聊天接口很多 IDE、旧插件、兼容层都还在用它。
/v1/responses新版统一响应接口新版 SDK 更常见。
/v1/embeddings向量接口用于知识库、RAG、语义检索。
/v1/images/generations图片生成用于海报、封面、视觉素材。
/v1/images/edits图片编辑通常需要原图或遮罩。
/v1/audio/transcriptions语音转文字常见于 Whisper 兼容接口。

5. Capabilities 是什么

前面的单项探测回答的是“这个接口能不能用”，而 capabilities 更像“细扫”：

  • 不是只看端点通不通，而是看哪个模型在这些端点上真的能用。
  • 很适合排查“模型名列出来了，但实际调用失败”这种第三方网关常见问题。
  • 开启后页面会额外生成一份完整报告，帮你总结推荐模型、已测通端点、失败端点和接入建议。

6. opencode / IDE 怎么填更稳

  • 最稳的填写方式通常是 Base URL 填到 /v1 为止。
  • 如果客户端支持自定义后缀，就按检测结果使用 /v1/chat/completions 或 /v1/responses。
  • 如果客户端固定某个后缀、又不允许修改，而网关不支持那个后缀，就会直接不兼容。
