  • 输入网关根地址，例如 `https://example.com`。

能力分别代表什么

  • models：列出这个网关暴露的模型名，先确认有哪些模型能被调。
  • chat_completions：最基础的文本对话能力，适合聊天、问答、代码辅助、文案生成。
  • tool_calling：函数/工具调用能力，适合多智能体编排、自动化工作流、外部工具接入。
  • responses：较新的统一接口能力，适合接新式 SDK 和更完整的 agent workflow。
  • embeddings：向量嵌入能力，适合 RAG、知识库检索、语义搜索。
  • images：图片生成接口，适合封面图、海报、配图、视觉原型。
  • docs：文档类端点是否存在，主要用于补充判断，不是核心能力。

这个 Key 建议用来做什么

  • 如果 `chat_completions` 通过：适合做普通聊天、文案生成、总结、代码解释。
  • 如果 `tool_calling` 也通过：适合做多智能体系统、自动化助手、任务编排平台。
  • 如果 `responses` 也通过：适合优先接新式 OpenAI SDK，后续升级空间更大。
  • 如果 `embeddings` 不通过：不建议直接拿它做 RAG 或知识库向量检索。
  • 如果 `images` 不通过：说明这个 Key 更适合文本任务，不适合图像生成场景。
  • 如果核心三项都通过：这是比较适合做 Agent 平台、研究工作台、自动化应用的 Key。
