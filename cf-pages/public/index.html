Gateway Capability Probe

输入 OpenAI-compatible 网关地址和 API Key，快速判断这个网关到底适合做聊天、工具调用、多智能体、RAG，还是只能做基础文本任务。

系统会依次检查 models、chat、tool calling、responses、embeddings、images 和 docs。

如何使用

  • 输入网关根地址，例如 `https://example.com`。
  • 输入对应 API Key。
  • 点击“开始检测”，等待结果返回。
  • 优先关注 `chat_completions`、`tool_calling`、`responses`，这三项最影响多智能体可用性。

每个能力对应什么

  • models：列出这个网关暴露了哪些模型，先判断模型可见范围。
  • chat_completions：最基础的文本能力，适合聊天、问答、总结、代码解释、文案生成。
  • tool_calling：工具调用能力，适合多智能体系统、自动化助手、工作流编排。
  • responses：较新的统一接口能力，适合接新版 SDK 和更完整的 agent workflow。
  • embeddings：向量嵌入能力，适合 RAG、知识库检索、语义搜索。
  • images：图片生成能力，适合海报、封面图、配图、设计草图。
  • docs：补充判断文档端点和接口实现风格，不是核心能力。

这个 Key 适合用来做什么

  • 如果 `chat_completions` 通过：适合普通文本任务，比如聊天、总结、写作、代码辅助。
  • 如果 `tool_calling` 也通过：适合做多智能体、自动化执行、外部工具接入。
  • 如果 `responses` 也通过：适合优先接新版 OpenAI SDK，后续扩展性更好。
  • 如果 `embeddings` 不通过：不建议直接拿它做 RAG 或向量数据库检索。
  • 如果 `images` 不通过：说明这个 Key 更偏文本用途，不适合图像生成场景。
  • 如果核心三项都通过：通常适合做 Agent 平台、自动化工作台、研究助手。
