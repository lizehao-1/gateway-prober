Gateway Capability Probe

输入 Base URL 和 API Key，快速判断这个网关更适合聊天、工具调用、Responses，还是只适合基础文本任务。

网页不会持久化保存你的 Key。
正在检测，请稍候。

默认重点检测 models、chat、tool calling、responses、embeddings。

如何理解结果

  • chat_completions 通过：说明传统聊天接口可用。
  • tool_calling 通过：更适合 agent、自动化、工具调用场景。
  • responses 通过：更适合新版 SDK。
  • embeddings 不通过：不适合知识库、RAG、语义检索。

如果 embeddings 不可用

  • 不影响普通聊天、代码问答、工具调用。
  • 会影响知识库问答、RAG、语义搜索、文档召回。
  • 如果你的 IDE 或工作流依赖向量检索，这部分体验会变弱，甚至不可用。

opencode / IDE 接入建议

  • 最稳的填写方式通常是 Base URL 填到 /v1 为止。
  • 如果客户端支持自定义后缀，就按检测结果使用 /v1/chat/completions 或 /v1/responses。
  • 如果客户端固定某个后缀、又不允许修改，而网关不支持那个后缀，就会直接不兼容。

常见后缀说明

  • /v1/chat/completions：传统聊天接口
  • /v1/responses：新版统一响应接口
  • /v1/embeddings：向量接口，用于知识库/RAG
  • /v1/images/generations：图片生成接口
